Why Use Prototypes?

  • Saves Memory:

    Shared methods are stored in one place (the prototype), not copied to every object.

  • Easy to Share Behavior:

    You can add new methods to the prototype, and all objects automatically get access.

Closures in JavaScript

A closure is when a function “remembers” the variables in the place where it was created, even if it’s used somewhere else. It’s like the function carries a secret backpack 🎒 full of the variables it might need.

The Basic Idea

A function inside another function can access:

  1. Its own variables.
  1. The variables of the function where it was created (the outer function).
  1. Global variables.

Example of a Closure

function outerFunction() {
    let secret = "I’m a secret!"; // Variable in the outer function

    function innerFunction() {
        console.log(secret); // Inner function uses the outer variable
    }

    return innerFunction;
}

const myClosure = outerFunction(); // outerFunction runs and returns innerFunction
myClosure(); // Output: I’m a secret!

What’s Happening?

  1. outerFunction runs: It creates the secret variable and the innerFunction.
  1. innerFunction is returned: Even though outerFunction is done, innerFunction keeps access to secret.
  1. Closure magic happens: myClosure (the returned innerFunction) remembers the secret variable because of the closure.

Why Are Closures Useful?

1. Private Variables

Closures let you keep data safe from the outside world.

function createSecret(secret) {
    return function () {
        console.log(`The secret is: ${secret}`);
    };
}

const mySecret = createSecret("I love chocolate!");
mySecret(); // Output: The secret is: I love chocolate!

The secret variable is only accessible inside the closure. It’s private!

2. Building Custom Functions

You can use closures to create functions with their own “settings.”

function multiplier(factor) {
    return function (number) {
        return number * factor;
    };
}

const double = multiplier(2); // Function to multiply by 2
const triple = multiplier(3); // Function to multiply by 3

console.log(double(5)); // Output: 10
console.log(triple(5)); // Output: 15

Each returned function remembers its own factor.

3. Making Counters

Closures let you keep track of things, like a counter.

function createCounter() {
    let count = 0;

    return function () {
        count++;
        console.log(`Current count: ${count}`);
    };
}

const counter = createCounter();
counter(); // Output: Current count: 1
counter(); // Output: Current count: 2
counter(); // Output: Current count: 3

The count variable is private but still remembered by the returned function.

Summary

  • Closure = Function + Backpack 🎒
  • A closure is a function that carries the variables it needs from its original environment.

Uses of Closures:

  1. Keep variables private.
  1. Create custom functions.
  1. Track data (like counters).

Event Delegation

Event delegation ek technique hai jisme hum:

  1. Ek single event listener parent element pe lagate hain.
  1. Aur usse child elements ke events handle karte hain (including future dynamically added elements).

Logic kaise kaam karta hai?

Iska base hai event bubbling, jisme event sabse pehle target element pe trigger hota hai aur phir parent elements tak "bubble up" karta hai.

Kyu use karte hain Event Delegation?

  1. Efficient hota hai:

    Har child pe alag-alag listener lagane ki zarurat nahi, ek hi parent pe kaafi hai.

  1. Dynamic elements ke liye perfect hai:

    Agar future mein naye elements add hue, toh unka event bhi automatically handle ho jayega.

Example Without Event Delegation

<div id="parent">
    <button>Button 1</button>
    <button>Button 2</button>
    <button>Button 3</button>
</div>

Agar har button ke liye alag listener lagate ho:

const buttons = document.querySelectorAll("button");
buttons.forEach((button) => {
    button.addEventListener("click", () => {
        console.log("Button clicked!");
    });
});

Problem:

  • Har button ke liye ek alag listener lag raha hai.
  • Dynamic buttons (jo baad mein add hote hain) kaam nahi karenge.

Event Delegation Ka Solution

Ab ek hi parent pe listener lagate hain:

const parent = document.getElementById("parent");

parent.addEventListener("click", (event) => {
    if (event.target.tagName === "BUTTON") {
        console.log("Button clicked!");
    }
});

Kaise Kaam Karta Hai?

  1. Listener sirf parent pe hai.
  1. Jab button click hota hai, event bubble hoke parent tak aata hai.
  1. event.target se pata chalta hai ki kaunsa element click hua.
  1. Condition lagate hain ki agar event.target button hai, toh kaam karo.

Advantages

  1. Kam Listeners:

    Har child element pe listener lagane ki zarurat nahi.

  1. Dynamic Elements ka Support:

    Agar naye buttons add karte ho, toh bhi woh automatically kaam karenge:

newButton.textContent = "New Button";
parent.appendChild(newButton); // Listener pehle se hi kaam karega!

HOFs

What are Higher-Order Functions?

Ek Higher-Order Function (HOF) wo function hota hai jo:

  1. Ek function ko argument ke tarah accept karta hai, ya
  1. Ek function ko return karta hai.

Bas itna yaad rakho ki agar function ke andar koi aur function kaam kar raha hai, toh wo HOF hai.

Why are HOFs Useful?

  • Reusable Code: Bar-bar repeat hone wale kaam ke liye ek generic function bana sakte ho.
  • Functional Programming ke concepts implement karte ho.

Example 1: HOF jo Function ko Accept Karta Hai

function greet(name, callback) {
    console.log("Hi " + name);
    callback();
}

function sayBye() {
    console.log("Bye!");
}

greet("Harsh", sayBye)

Explanation:

  1. greet ek HOF hai kyunki wo ek function (callback) ko argument ke tarah accept kar raha hai.
  1. greet pehle name ko greet karta hai aur phir callback function (sayBye) ko call karta hai.

Example 2: HOF jo Function Return Karta Hai

function multiplier(factor) {
    return function (number) {
        return number * factor;
    };
}

const double = multiplier(2);
console.log(double(5)); // Output: 10

Explanation:

  1. multiplier ek HOF hai kyunki wo ek function return kar raha hai.
  1. double ab ek specific function hai jo kisi number ko 2 se multiply karega.

Common HOFs in JavaScript

JavaScript mein kuch built-in HOFs hain jo bohot common aur powerful hain. Let’s see unka use.

1. forEach

Har array element pe kaam karne ke liye.

const arr = [1, 2, 3];
arr.forEach((num) => console.log(num * 2));

Output:

2

4

6

2. map

Naye array banane ke liye.

const arr = [1, 2, 3];
const squares = arr.map((num) => num * num);
console.log(squares); // Output: [1, 4,

3. filter

Array ke specific elements ko filter karne ke liye.

const arr = [10, 15, 20, 25];
const filtered = arr.filter((num) => num > 15);
console.log(filtered); // Output: [20, 25]

4. reduce

Array ko reduce karke ek single value banane ke liye.

const arr = [1, 2, 3, 4];
const sum = arr.reduce((acc, num) => acc + num, 0);
console.log(sum); // Output: 10

Real-Life Analogy

Socho ek chef hai jo ek universal recipe follow karta hai:

  1. Tum ek ingredient list dete ho (function as input).
  1. Chef tumhare hisaab se dish ready karta hai (function output deta hai).

Chef ka recipe-making logic ek HOF ki tarah kaam kar raha hai.

Key Points to Remember

  • HOFs are functions that accept or return other functions.
  • They help you write short, clean, and reusable code.
  • JS ke built-in HOFs like map, filter, reduce bohot kaam ke hain.

Error handling

Error Handling in JavaScript

Code likhte waqt kabhi-kabhi errors aajate hain (syntax error, runtime error, etc.). Agar in errors ko handle nahi kiya, toh poora code crash kar sakta hai.

Error handling ka main purpose hai:

  1. Errors ka pata lagana (detect).
  1. Unhe theek se manage karna (handle) taaki code crash na ho.

Try-Catch Block
JavaScript mein, errors ko handle karne ke liye hum try-catch ka use karte hain.
Iska structure kuch aisa hota hai:

try {
    // Risky code jo error throw kar sakta hai
} catch (error) {
    // Agar error aata hai toh yeh block execute hoga
}

Basic Example

try {
    let result = 10 / 0; // Risky code
    console.log(result); // This runs fine
    console.log(a); // Error: 'a' is not defined
} catch (error) {
    console.log("An error occurred:", error.message);
}

Explanation:

  • Try block: Yaha wo code likhte ho jo error throw kar sakta hai.
  • Catch block: Agar try block mein koi error aata hai, toh catch block execute hota hai.
  • error.message: Error ka specific message batata hai.

Code crash nahi karta, error handle ho jata hai.

Finally Block
Agar tumhe koi code try-catch ke baad bhi hamesha execute karna ho, toh finally use karte hain.

try {
    let result = 5 / 1;
    console.log(result);
} catch (error) {
    console.log("An error occurred:", error.message);
} finally {
    console.log("This runs no matter what.");
}

Output:

5
This runs no matter what.

Throwing Your Own Errors
Kabhi-kabhi tumhe khud error throw karni padti hai (e.g., input validation).

function divide(a, b) {
    if (b === 0) {
        throw new Error("Cannot divide by zero!"); // Custom error
    }
    return a / b;
}

try {
    console.log(divide(10, 0));
} catch (error) {
    console.log("Error:", error.message);
}

Real-Life Analogy
Imagine ek ATM hai. Agar tum wrong PIN enter karte ho:

  • ATM tumhe error message (catch block) deta hai: "Wrong PIN entered."
  • Lekin ATM shutdown nahi karta, baaki cheezein theek kaam karti rehti hain.

Similarly, try-catch se tumhara code rukta nahi hai, error handle ho jata hai.

When to Use Try-Catch?

  1. API Calls: Jab tum external APIs se data fetch karte ho.
  1. File Operations: Jab tum file read/write karte ho.
  1. Validation: Kabhi input ya calculation galat ho sakti hai.
  1. Complex Logic: Jahaan code fail hone ke chances zyada hain.

Golden Rule

  • Try-catch ka use tab karo jab error expected ho, har chhoti cheez ke liye mat lagao.
  • Agar error predictable hai, toh uska check (if-else) karna better hota hai.

Custom Events in JavaScript

What are Custom Events?
Jaise JavaScript mein predefined events hote hain (e.g., click, keydown), waise hi tum custom events create kar sakte ho. Custom events ka use tab hota hai jab tumhe apne specific app logic ke liye events banane aur unhe trigger karne ki zarurat ho.

Why Use Custom Events?

  1. Communication:
    • Different parts of your app ek dusre se baat kar sakte hain.
  1. Reusable Logic:
    • Ek baar custom event bana liya, toh kahi bhi use kar sakte ho.
  1. Dynamic Functionality:
    • User actions ya background processes ke response mein kuch specific karna ho.

How to Create and Use Custom Events?

1. Custom Event Create Karna
JavaScript mein
CustomEvent constructor ka use karke custom events bana sakte ho.

const myEvent = new CustomEvent("greet", {
    detail: { name: "Harsh", message: "Hello, world!" }
});
  • "greet": Event ka naam hai.
  • detail: Ek object hai jo tumhare custom data ko carry karta hai.

2. Event Listen Karna
Custom events ko listen karne ke liye hum
addEventListener ka use karte hain.

document.addEventListener("greet", (event) => {
    console.log("Custom Event Triggered!");
    console.log(event.detail); // Output: { name: "Harsh", message: "Hello, world!" }
});

3. Event Trigger Karna
Custom events ko trigger karne ke liye hum
dispatchEvent ka use karte hain.

document.dispatchEvent(myEvent);

Output:

Custom Event Triggered!
{ name: "Harsh", message: "Hello, world!" }

Full Example:

// Custom Event Create karo
const userLoginEvent = new CustomEvent("userLogin", {
    detail: { username: "harsh123", time: "10:30 AM" }
});

// Event Listener Add karo
document.addEventListener("userLogin", (event) => {
    console.log("User Logged In!");
    console.log("Username:", event.detail.username);
    console.log("Login Time:", event.detail.time);
});

// Event Trigger karo
document.dispatchEvent(userLoginEvent);

Output:

User Logged In!
Username: harsh123
Login Time: 10:30 AM

Real-Life Analogy:
Imagine tumhare ghar pe ek party hai:

  1. Custom Rule:
    • "Jab koi bell bajaye, toh welcome bolo."
  1. Listener:
    • Family members bell bajne ka wait kar rahe hain.
  1. Trigger:
    • Guest bell bajata hai (event trigger karta hai).
  1. Result:
    • Sab milke "Welcome!" bolte hain. 😄

Where Are Custom Events Used?

  1. Web Components:
    • Reusable components ke beech communication ke liye.
  1. Real-Time Updates:
    • E.g., live chat apps mein user typing status dikhana.
  1. Games:
    • Score updates ya level completion ke liye.
  1. Custom Libraries:
    • Apne framework ya library mein events implement karne ke liye.
